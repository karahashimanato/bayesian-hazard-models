# bayesian-survival-analysis

PyMCを用いたベイズ生存時間分析の実装・評価比較リポジトリ。

Telco Customer Churn（顧客解約）データを題材に、統計的な「推定・解釈」を目的とした基礎的なハザードモデルの構築から、最新のPyMC 6 / ArviZ環境を用いた「予測性能の検証」および「個別顧客の生存シミュレーション（ビジネス活用）」までを網羅した包括的なベイズ生存時間分析パイプラインを実装しています。

---

## Overview

生存時間分析は「イベント（解約）が起きるまでの時間」を扱う統計手法群です。観測データの一部が**右側打ち切り（right censoring）**——観測終了時点でまだイベントが起きていない対象は、真のイベント時刻ではなく「少なくともここまでは生存していた」という下限情報しか持たない——という特徴を持ちます。

本プロジェクトでは、この打ち切りを尤度に正しく組み込んだ上で、5つの評価指標（C-index, Time-Dependent AUC, Brier Score, LOO-CV, 予測生存曲線）を用いてモデルの汎化性能を検証し、科学的な意思決定に接続するアプローチを確立しています。

---

## データ

* **出典**: IBM Sample Data — Telco Customer Churn（7,043顧客）
* **対象変数**: `tenure`（在籍月数）、`Churn`（解約有無）
* **データ分割**: 
  * 訓練（80%）とテスト（20%）に分割。
  * `event` 列（解約の有無）で**層別化（`stratify`）**を行うことで、訓練データ・テストデータの解約率を完全に一致（共に **26.54%**）させ、学習と評価の偏りを排除。

---

## モデル一覧

本リポジトリでは、制約の強いパラメータモデルから、共変量やグループ変量効果（Frailty）を取り入れた柔軟な非比例ハザード寄りのモデルまで段階的に拡張しています。

| モデル | ハザード関数 $h(t)$ | 特徴 |
| :--- | :--- | :--- |
| **Exponential** | $h(t) = \lambda$ (時間一定) | 最もシンプル。3モデル中もっとも制約が強い。 |
| **Weibull** | $h(t) = \frac{k}{\lambda} \left(\frac{t}{\lambda}\right)^{k-1}$ | 形状パラメータ $k$ でハザードの時間変化（増加/減少）を表現。 |
| **Cox比例ハザード**<br>*(Piecewise Exponential)* | $h(t\|x) = h_{0, k(t)} \times \exp(x^T \beta)$ | 6ヶ月の区間ごとに自由なベースラインハザードを推定。 |
| **Cox Frailtyモデル**<br>*(支払方法の変量効果)* | $h(t\|x) = h_{0, k(t)} \times \exp(x^T \beta) \times z_{\text{payment}}$ | 支払方法（4グループ）ごとに、平均1に制約された変量効果 $z$（Frailty）を導入。 |

* ※いずれも尤度は `pm.Potential` や `pm.Deterministic` を用いて、右側打ち切りを考慮した統一対数尤度 $event_i \log h(t_i) + \log S(t_i)$ を直接実装。

## 主要な結果

### 1. 統計的推定と仮説検証

#### ■ Exponentialモデル
* $\lambda \approx 0.00838$（平均解約期間 $1/\lambda \approx 119$ヶ月）
* Kaplan-Meier曲線との比較で、**序盤（0-10ヶ月）の急減衰・終盤（40-72ヶ月）の緩やかな減衰**を全く再現できず、ハザード率一定という仮定が実データと整合しないことが判明。

#### ■ Weibullモデル
* $k \approx 0.692$ (89%確信区間 `[0.67, 0.71]`) — **1を明確に下回り、ハザードが時間とともに減少する**（在籍が長いほど解約しにくくなる）性質を定量的に確認。
* Kaplan-Meier曲線とほぼ完全に一致し、Exponentialで見られた系統的ズレが解消。

#### ■ Cox比例ハザード (Piecewise Exponential)
* 区間ごとのベースラインハザード $h_0$ が、時間の経過とともに減少（`0.197` [0-6ヶ月] $\to$ `0.073` [60-66ヶ月]）することを確認し、Weibullモデルの発見と独立に整合。
* 契約タイプの効果（月間契約を基準とした比較）：
  * **1年契約**: $\beta \approx -1.017$（ハザード比 $\exp(\beta) \approx 0.362$ / 月間契約よりハザード約64%減）
  * **2年契約**: $\beta \approx -2.477$（ハザード比 $\exp(\beta) \approx 0.084$ / 月間契約よりハザード約92%減）

### 2. 予測性能評価とモデル比較 (Held-outデータによる検証)

訓練に含めない完全なテストデータ（Held-out）を用いて、各モデルの予測性能を多角的に評価しました。

#### ■ 総合評価テーブル

| モデル構成 | C-index（順位付け） | Brier Score（確率精度） | 平均時間依存性 AUC | LOO (elpd_diff) | 有効パラメータ数 ($p_{\text{LOO}}$) |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **ナイーブ基準（全員同一確率）** | 0.500 | ≈0.197 | 0.500 | - | - |
| **Contract のみ (Cox model)** | 0.767 | 0.305 | - | -60.0 | 10.2 |
| **Contract + Payment (Frailty)** | **0.785** | **0.303** | **0.844** | **0.0 (基準)** | **12.3** |
| **上記 ＋ 3ヶ月刻み (24区間)** | 0.764 | 0.458 | - | - | - |

#### ■ 予測性能から得られた重要な洞察

1. **LOOが示す「Frailtyモデル」の圧倒的優位性**
   LOOによるモデル比較の結果、支払方法（PaymentMethod）の変量効果を加えたFrailtyモデルは、通常のCoxモデルに対して $elpd\_diff = -60.0$（標準誤差 $dse = 10.0$）という大差をつけて勝利（差は標準誤差の**6倍**）。ペナルティとなる有効パラメータ数の増加はわずか `2` 程度に抑えられており、階層構造（変量効果）が過学習を高度に抑制しながら予測力を高めていることが証明されました。
2. **C-index と Brier Score の独立性**
   変数を追加したことで順位付け（C-index）は大きく向上しましたが、確率の絶対的なズレ（Brier Score）はほぼ変化しませんでした。「誰が先に解約するか（順位）」の予測力向上と、「何%で解約するか（絶対確率）」の較正（キャリブレーション）は全く別問題であることを実証しました。
3. **区間の細分化による「過学習」の実例**
   時間区間を12 $\to$ 24区間に倍増させたところ、区間あたりのサンプル数が極小化し、訓練データのノイズを過剰に学習。C-index、Brier Scoreともに大幅に悪化し、Held-out評価の重要性を体現する結果となりました。
4. **時間とともに予測精度が上がる「時間依存性AUC」**
   平均時間依存性AUCは **0.844** と極めて高い精度を記録。さらに、契約初期（6ヶ月目）のAUC `0.78` に対し、契約経過に伴い精度は右肩上がりに上昇、**30ヶ月経過以降は `0.95`（ほぼ完璧な予測）** に到達。ビジネスにおいて「失うと最も痛手となる中長期ロイヤル顧客の離脱予兆」をピンポイントかつ極めて正確に捉えられるモデルであることが示されました。

### 3. 個別顧客の生存曲線シミュレーション

構築した最良モデル（Frailty）を用いて、テストデータから抽出した代表的な顧客プロファイルごとの未来の生存確率をシミュレーションしました。

* **高リスク（1ヶ月契約 / 電子チェック払い）**: 1年後の生存確率は約50%、2年後には20%未満へと急激に低下。早期のプラン変更や自動決済移行プロモーションの対象。
* **中リスク（1年契約 / クレジットカード自動決済）**: 1年・2年といった契約更新タイミング（12ヶ月、24ヶ月付近）で生存確率が階段状に「カクン」と下がるハザード特性を正確にシミュレート。
* **低リスク（2年契約 / クレジットカード自動決済）**: 3年経過時点でも85%以上の生存確率を維持。解約防止施策は不要で、アップセル推奨ターゲット。

* ※**モデルの数学的性質**: 今回は離散共変量のみを用いているため、顧客全体の生存曲線は実質的に **12パターン** に集約されます。すべての生存曲線は共通のベースラインハザード $S_0(t)$ を個人のハザード比 $HR_i$ でべき乗したもの（$S_i(t) = [S_0(t)]^{HR_i}$）になるため、ハザード比の高さによって減少速度は違えど、「減少する（カクンと落ちる）波のタイミング」は全員でシンクロします。

## 診断・実装における重要ハック

最新のPyMC 6 / ArviZ環境、および高速なNumPyroサンプラーを使用する上で生じる、JAXやライブラリの仕様によるエラーを以下のテクニックで回避しています。

* **対数尤度の個別保存**: `pm.Potential` では対数尤度グループが自動保存されないため、各データ点の対数尤度を `pm.Deterministic("lik_i", log_lik_elementwise)` で明示的に保存。
* **JAX Array の不変性（Immutable）エラー回避**: NumPyroが生成する対数尤度はJAX配列であり、ArviZのPSIS-LOO（パレート平滑化）時のインプレース上書きと衝突しクラッシュします。そのため、`np.asarray()` を用いて通常のNumPy配列にキャストして登録するハックを実装。
* **DataTree へのダイレクト代入**: ArviZ 0.20+以降の `xarray.DataTree` 構造に対応するため、旧式の `add_groups` ではなく、`idata["log_likelihood"] = ...` のように直接Datasetを代入。
* **IPCW（逆確率重み付け）のゼロ除算回避**: 最大生存時間（72.5ヶ月）に生存者が集中し、打ち切り生存確率 $G(72.5) = 0$ になることで、Brier/AUC計算時にゼロ除算エラーが発生。テストデータの最大時間を **72.4ヶ月** に極わずかにクリップすることで、評価精度を損なわずにエラーを完璧に解決。
