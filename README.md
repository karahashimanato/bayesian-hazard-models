# bayesian-survival-analysis

PyMCを用いたベイズ生存時間分析の実装比較。Telco Customer Churn(顧客解約)データを題材に、**Exponential → Weibull → Cox比例ハザード(Piecewise Exponential近似)**の3モデルを段階的に構築し、それぞれの前提(ハザード率が時間一定か、時間変化するか、共変量の効果をどう組み込むか)がデータの当てはまりにどう影響するかを検証する。

## Overview

生存時間分析は「イベントが起きるまでの時間」を扱う統計手法群で、観測データの一部が**右側打ち切り(right censoring)**——観測終了時点でまだイベントが起きていない対象は、真のイベント時刻ではなく「少なくともここまでは生存していた」という下限情報しか持たない——という特徴を持つ。本リポジトリでは、この打ち切りを尤度に正しく組み込んだ上で、3種類のハザードモデルを比較する。

## データ

- **出典**: IBM Sample Data — Telco Customer Churn(7,043顧客)
- **対象変数**: `tenure`(在籍月数)、`Churn`(解約有無)
- **打ち切りの構造**: `Churn=Yes`(1,869人)はイベント発生(解約)、`Churn=No`(5,174人)は観測終了時点でまだ在籍中の打ち切りデータ
- 前処理: `event`列(0/1)、`Contract`のダミー変数(`is_one_year`, `is_two_year`、基準カテゴリ=Month-to-month)を追加

## モデル一覧

| モデル | ハザード関数 h(t) | 特徴 |
|---|---|---|
| **Exponential** | h(t)=λ (時間一定) | 最もシンプル。3モデル中もっとも制約が強い |
| **Weibull** | h(t)=(k/λ)(t/λ)^(k-1) | 形状パラメータkでハザードの時間変化(増加/減少)を表現 |
| **Cox比例ハザード(Piecewise Exponential近似)** | h(t\|x)=h_{0,k(t)}×exp(x^T β) | 区間ごとに自由なベースラインハザード+共変量の比例効果 |

いずれも尤度は`pm.Potential`で直接実装（打ち切りを考慮した統一対数尤度: event_i * log h(t_i) + log S(t_i)）。

## 主要な結果

### 1. Exponentialモデル

- λ ≈ 0.00838 (平均解約期間 1/λ ≈ 119ヶ月)
- Kaplan-Meier曲線との比較で、**序盤(0-10ヶ月)の急減衰・終盤(40-72ヶ月)の緩やかな減衰**を再現できず、系統的なズレが残った
- ハザード率一定という仮定が、実データの構造と整合しないことが判明

### 2. Weibullモデル

- k ≈ 0.692 (89%区間[0.67, 0.71]) — **1を明確に下回り、ハザードが時間とともに減少する**ことを定量的に確認
- λ ≈ 198.9
- Kaplan-Meier曲線とほぼ完全に一致。Exponentialで見られた系統的ズレが解消

### 3. Cox比例ハザード(Piecewise Exponential)

- ベースラインハザードh0が6ヶ月区間ごとに、**0.197(0-6ヶ月)→0.073(60-66ヶ月)と明確に減少** — Weibullのk<1という発見と独立に整合
- 契約タイプの効果(月間契約を基準とした比較):

| 契約タイプ | β(対数ハザード比) | ハザード比 exp(β) | 解釈 |
|---|---|---|---|
| 1年契約 | -1.017 | ≈0.362 | 月間契約よりハザード約64%減 |
| 2年契約 | -2.477 | ≈0.084 | 月間契約よりハザード約92%減 |

「月間契約の方が解約しやすい」という仮説が、強く定量的に裏付けられた。

## 診断上のポイント

- 3モデルすべてでdivergences=0、r_hat=1.00の健全なサンプリングを達成
- Coxモデルはess_bulkが8,000〜9,000超(理論上限の8,000近辺)という極めて高い効率を記録。これは各区間のベースラインハザードパラメータが互いに比較的独立(強い相関構造を持たない線形的な構造)であるため、NUTSが効率的に探索できたことによる。SIRS/SEIRのような時間発展を通じてパラメータが複雑に絡み合うモデルとの対比が興味深い。

## 事前分布設計における学び

- **Prior predictive checkでcensoringを考慮しない単純比較は誤解を招く**: 全データ(打ち切り含む)のヒストグラムとシミュレーションを素朴に比較すると、打ち切りにより長期在籍者が偏って蓄積された二峰性が現れ、誤ったモデル診断につながる。`event=1`のみに絞った比較が必要。
- **サブグループ(契約タイプ別)の事前分布は、全体統計から機械的に流用すると外れる**: Coxモデルのベースラインハザードを全顧客の平均から設定すると、最も解約しやすいサブグループ(月間契約)の実態から大きく乖離した。対象とするサブグループに応じて、逆算の母集団を絞り込む必要がある。
- **分散(alpha)を締めすぎると、モデルが表現すべき区間間のばらつきをデータに語らせる余地を奪う**: h0の事前分布のalphaを50→10に緩めたことで、時間変化するハザードという構造をモデルが正しく捉えられるようになった。
